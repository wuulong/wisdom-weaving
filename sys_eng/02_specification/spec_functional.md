# 02. 功能性規格定義 (Functional Specification)

本文件將 `01_requirements/req_vision.md` 中定義的核心需求，轉化為系統必須實現的**具體功能規格**。

---

## 1. 系統邊界與主要模組 (System Boundary & Modules)

本系統為通用 Wisdom Weaving 智慧工程沙盒框架。核心在於透過標準的 Agent 提問與歸納機制產製結構化知識體系，並對使用者提問進行系統性回答。

* **主要功能模組**：
  - **提問與歸納萃取模組**：控制 Agents 相互問答並歸納出結構化知識。
  - **回答拓本與分發模組**：提供智慧運用者架構性的框架進行系統性探索。
  - **單一 SQLite 複合資料庫與語意層模組**：結合關聯表 (JSON Metadata)、向量欄位、與 ER 關係圖譜表。
  - **反思與聯網查詢模組**：在問答互動過程中反思盲點，聯網引入細緻理論補強。
  - **多維度向量空間與檔案寫入模組**：自動產製層級化 Markdown 檔案。
  - **CLI / Skill 初始建置與運行模組**：提供文本引導、系統引導與持續運行的控制入口。
  - **釋出合規與文本處理工具集**：負責模擬文本生成，以及發布時的 Layer 1 原始文本剝離。

---

## 2. 功能規格列表 (Functional Specs List)

* **`[SPC-001]` Agent 提問與歸納知識萃取規格**
  - **追溯鏈**：`[追溯：REQ-003]`
  - **功能描述**：系統應定義「提問 Agent」與「回答/翻譯 Agent」之問答協定。提問 Agent 掃描文本並發起跨領域提問，歸納 Agent 監控對話並從中萃取出結構化知識。
  - **處理邏輯 (Process Logic)**：
    1. 提問 Agent 發起具備好奇心/跨領域視角的提問（例如：《鹿鼎記》中的多重身份衝突）。
    2. 回答 Agent 調用背景知識進行解答。
    3. 歸納 Agent 利用長上下文窗口（Long Context Window）將問答過程進行聚類分析，歸納出結構化 JSON。

* **`[SPC-002]` 負責回答 the Agent 拓本架構與查詢分發規格**
  - **追溯鏈**：`[追溯：REQ-003]`
  - **功能描述**：系統應建立負責回應智慧運用者的 Agent 拓本結構。該結構引導智慧運用者遵循架構性框架（而非隨機盲目提問）對知識體系進行系統性探索。
  - **處理邏輯 (Process Logic)**：
    1. 接收智慧運用者的提問。
    2. 依據知識體系目錄，將提問分發至對應層級的負責 Agent。
    3. 負責 Agent 循架構樹回傳系統性、關聯性的解答。

* **`[SPC-003]` 關聯式資料庫與 JSON Metadata 規格**
  - **追溯鏈**：`[追溯：REQ-001]`
  - **功能描述**：系統應能針對不同來源的文本，動態建立與初始化對應的獨立資料庫檔案，在單一 SQLite 資料庫內建置核心儲存，並搭配以 JSON 為內容的 Metadata 結構，用以儲存與維護關聯性數據（如章節存取、實體 Entity 被何處參考、章節摘要等）。
  - **處理邏輯 (Process Logic)**：
    1. 根據輸入文本來源，動態路由與建立對應的獨立 SQLite 檔案（如 `ludingji.db`），實現多資料庫環境隔離。
    2. 建立章節結構表、Entity 對照表與引用映射表。
    3. 在相關欄位中利用 JSON 格式寫入富 Metadata（包含 Entity 的特徵、出現頻率與章節摘要）。
    4. 支援查詢「特定實體被哪些章節或段落參考」等結構化查詢。

* **`[SPC-004]` 向量資料庫與語意查詢規格**
  - **追溯鏈**：`[追溯：REQ-001]`
  - **功能描述**：系統應在同一個 SQLite 資料庫內，將文本切片及萃取知識進行向量化 Embeddings 存儲，提供語意相似度檢索（藉由載入 sqlite-vss 或等同向量模組）。
  - **處理邏輯 (Process Logic)**：
    1. 對小說/文本切片或已產製的知識節點生成向量。
    2. 提供語意查詢接口，接收語意 Query 並檢索出最相關的上下文片段。

* **`[SPC-005]` ER 關係與語意圖譜建置與維護規格**
  - **追溯鏈**：`[追溯：REQ-002]`
  - **功能描述**：系統應在同一個 SQLite 資料庫中建立實體關係表 (`entity_relations`)，定義角色、組織、身分、道具等多維關係，而無需額外使用外部圖資料庫（如 Neo4j）。
  - **處理邏輯 (Process Logic)**：
    1. 定義節點 (Node) 類型與關係 (Relation) 類型。
    2. 提供圖譜維護 API（新增節點、更新關係屬性如「恩情值」、「敵對值」）。
    3. 支援 SQL 遞迴查詢 (Common Table Expressions, CTE)，提供 Agent 高維度的地緣政治與人際關係推理上下文。

* **`[SPC-006]` 問答觸發知識缺漏反思與聯網查詢規格**
  - **追溯鏈**：`[追溯：REQ-004]`
  - **功能描述**：系統在提問與回答的過程中，應能主動發現知識體系的缺漏與邏輯盲點。在回答時若需要聯網獲取更多基礎資訊，應能自動調用聯網查詢，使更細膩與真實的資訊進到知識系統中以完成補強。
  - **處理邏輯 (Process Logic)**：
    1. 在問答對話流中，若回答 Agent 感知當前 Context 存在知識空缺或不確定性。
    2. 回答 Agent 自動觸發聯網查詢（Google Search API）獲取基礎細節與外部文獻。
    3. 擷取聯網資訊並將更細緻的內容增量融入知識體系，使遺漏的知識在問答中動態補充。

* **`[SPC-007]` 多維度向量空間建置與 Markdown 檔案寫入規格**
  - **追溯鏈**：`[追溯：REQ-005]`
  - **功能描述**：系統應將萃取出的知識體系映射至多維度向量空間，並自動在本地端目錄中產製結構化、符合 Markdown 規範的目錄與檔案。
  - **處理邏輯 (Process Logic)**：
    1. 解析知識體系 JSON，轉換為層級化的目錄結構。
    2. 生成包含 Meta metadata、章節摘要與 Entity 關聯說明的 Markdown 文件。
    3. 寫入本地對應專案目錄。

* **`[SPC-008]` 系統起始建置與持續運行 Skill 規格**
  - **追溯鏈**：`[追溯：REQ-006]`
  - **功能描述**：系統應提供專屬 Skill，用於在新文本提供時自動觸發整個系統的初始化，並在使用者提出詢問（即將要知道）時，動態建置與演進知識體系。
  - **處理邏輯 (Process Logic)**：
    1. 使用者提供文本並調用 `wisdom-weaving init`。
    2. Skill 自動執行資料庫建置、向量化與 Agent 拓本對合。
    3. 依循 Just-In-Time 運作原則，於使用者提問時觸發動態對話建置更新。

* **`[SPC-009]` 統一入口 CLI 與自動化腳本規格**
  - **追溯鏈**：`[追溯：REQ-007]`
  - **功能描述**：為提升精準度、效率與減少 token，所有的資料庫讀寫、關聯性查詢與檔案產製必須優先由 Python 腳本執行，並透過統一的入口級 CLI 供使用者呼叫，且配備完整手冊。指令與運作必須支援指定資料庫引數運行。
  - **處理邏輯 (Process Logic)**：
    1. 開發 CLI 工具，作為使用者的唯一操作入口，全域支援 `--db` 或 `-d` 參數以指定目標 SQLite 資料庫。
    2. 將複雜的商業邏輯與檢索封裝為獨立的 Python 腳本（遵循 CLI/API 雙向相容架構，接收動態資料庫路徑或連線引數）。
    3. 每個腳本在腳本目錄下均有標準 Markdown 使用手冊說明書。

* **`[SPC-010]` 輕量核心 POC 限制與演進架構規格**
  - **追溯鏈**：`[追溯：REQ-008]`
  - **功能描述**：系統在設計與實作時，應維護一個最精簡的核心 POC。即使同樣是《鹿鼎記》，可以採用「某一個特定章節」或「某個特定主題的探索」來作為 POC 的核心範疇，確認穩定後方可擴展至整體。
  - **處理邏輯 (Process Logic)**：
    1. 限制核心 POC 的文本處理範圍至單一章節或特定主題。
    2. 確保核心組件（RAG 檢索、問答、歸納）高度內聚，修改邊界清晰。

* **`[SPC-011]` 模擬實驗文本設計與生成規格**
  - **追溯鏈**：`[追溯：REQ-009]`
  - **功能描述**：系統應設計與生成一組適合進行多代理人權謀問答實驗的無版權模擬文本，模擬多重衝突身份、關係拓本與動機對撞。
  - **處理邏輯 (Process Logic)**：
    1. 撰寫一組無版權糾紛的範本文字（例如：設定虛擬朝廷與江湖勢力）。
    2. 在該範本中人為埋入多個「知識盲點」與「資訊不對稱細節」。
    3. 用作 POC 階段的標準輸入文本，隨專案公開發佈。

* **`[SPC-012]` 資料庫 Layer 1 版權文本剝離與還原重建工具規格**
  - **追溯鏈**：`[追溯：REQ-010]`
  - **功能描述**：系統在發佈資料庫時，應能一鍵抹除 `contents` 表中的 `raw_text` 以保護版權，並提供本地端還原對齊的 CLI 工具。
  - **處理邏輯 (Process Logic)**：
    1. 抹除邏輯：執行 SQL `UPDATE contents SET raw_text = '';` 抹除原始段落文字，但保留 IDs 與向量 (Vector) 屬性。
    2. 還原邏輯：提供 `wisdom-weaving restore --text <local_path_to_ludingji>` 指令，讀取本地原著小說，按原本的字元長度/段落結構，將原始文字重寫回 `contents.raw_text`，完成 Layer 1 地端重建。
